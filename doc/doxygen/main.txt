/*!
\ingroup devel_doc
\defgroup error_management Exception management and translation

One of the major tasks for the Python wrapper of \c libpnicore is to translate
\c libpnicore's C++ exceptions into Python exceptions. 
There are two strategies how to do this job and both of them are used 

 - some of the C++ exceptions are translated into native, already existing
   Python exceptions

 - for the second part, new Python exceptions are created and exposed via the
   pni.core Python package.

The following list gives an overview how each C++ exception is treated 

C++ exception (pni::core::) | Python 2 exception          | Python 3 exception 
--------------------------- | --------------------------- | --------------------------
memory_allocation_error     | MemoryError                 | MemoryError
memory_not_allocated_error  | MemoryError                 | MemoryError
shape_mismatch_error        | pni.core.ShapeMismatchError | pni.core.ShapeMismatchError 
size_mismatch_error         | pni.core.SizeMismatchError  | pni.core.SizeMismatchError
index_error                 | IndexError                  | IndexError
key_error                   | KeyError                    | IndexError
file_error                  | pni.core.FileError          | pni.core.FileError 
type_error                  | TypeError                   | TypeError
value_error                 | ValueError                  | ValueError
range_error                 | pni.core.RangeError         | pni.core.RangeError 
not_implemented_error       | NotImplementedError         |  NotImplementedError
iterator_error              | pni.core.IteratorError      | pni.core.IteratorError
cli_argument_error          | pni.core.CliArgumentError   | pni.core.CliArgumentError
cli_error                   | pni.core.CliError           | pni.core.CliError

Those Python exceptions with a `pni.core` prefix are new exceptions 
provided by this wrapper. 
*/
